# Android Flaw - Backend Server

Backend server for the Android Flaw learning platform, providing AI-powered plan generation and Supabase database integration.

## Quick Start

### Prerequisites

- Node.js >= 22.11.0
- Supabase account (free tier works)
- OpenAI API key (optional, for AI plan generation)

### Setup Steps

#### 1. Install Dependencies

```bash
cd server
npm install express
```

#### 2. Configure Environment Variables

```bash
# Copy the example environment file
cp ../.env.example ../.env

# Edit .env with your values
```

**Required:**
- `SUPABASE_URL` - Your Supabase project URL
- `SUPABASE_ANON_KEY` - Your Supabase anonymous key

**Optional:**
- `OPENAI_API_KEY` - For AI-powered plan generation
- `PORT` - Server port (default: 8787)

#### 3. Set Up Database

1. Go to your Supabase Dashboard
2. Navigate to **SQL Editor**
3. Copy the contents of `schema.sql`
4. Paste and run the SQL script

This creates:
- User profiles table
- Skills catalog (15 pre-seeded skills)
- Learning content tables
- Progress tracking tables
- Achievements system
- Row Level Security policies

#### 4. Start the Server

```bash
# From project root
npm run api

# Or directly
cd server
node index.js
```

Server runs on `http://localhost:8787`

## API Endpoints

### `POST /generate-plan`

Generates a weekly learning plan for a skill.

**Request Body:**
```json
{
  "skillId": "uuid",
  "skillName": "Musique",
  "difficulty": "facile",
  "level": 1,
  "progress": 0,
  "weeklyTimeBudgetMinutes": 240,
  "tasks": [
    {
      "id": "task-1",
      "title": "Practice scales",
      "duration": "20 min",
      "xp": 50,
      "completed": false
    }
  ]
}
```

**Response:**
```json
{
  "skillId": "uuid",
  "difficulty": "facile",
  "sessionsPerWeek": 3,
  "weeklyTime": "1h30",
  "recommendedTaskIds": ["task-1", "task-2"],
  "sessions": [
    {
      "dayLabel": "Lun",
      "taskId": "task-1",
      "taskTitle": "Practice scales",
      "duration": "20 min",
      "xp": 50
    }
  ],
  "summary": "Plan facile pour Musique: 3 seances.",
  "source": "ai"
}
```

**Sources:**
- `ai` - Generated by OpenAI
- `fallback` - Generated by local algorithm (when no API key)

### `GET /health`

Health check endpoint.

**Response:**
```json
{ "ok": true }
```

## AI Plan Generation

### How It Works

1. **With OpenAI API Key:**
   - Sends structured prompt to GPT-4.1-mini
   - Receives JSON-formatted learning plan
   - Validates plan against available tasks

2. **Without API Key (Fallback):**
   - Uses built-in algorithm
   - Distributes tasks across week based on difficulty
   - Ensures balanced workload

### Difficulty Levels

| Level    | Sessions/Week | Estimated Time |
|----------|---------------|----------------|
| facile   | 3             | ~1-2 hours     |
| moyen    | 5             | ~2-3 hours     |
| difficile| 7             | ~3-5 hours     |

## Database Schema

### Core Tables

| Table | Purpose |
|-------|---------|
| `profiles` | User accounts and preferences |
| `skills_catalog` | Available skills to learn (15 pre-seeded) |
| `user_skills` | Skills user is learning |
| `learning_content` | Lessons, exercises, quizzes |
| `user_progress` | Individual lesson completion |
| `weekly_plans` | Generated weekly plans |
| `plan_sessions` | Sessions within plans |
| `session_runs` | Individual session tracking |
| `achievements` | Available achievements |
| `user_achievements` | User's unlocked achievements |

## Local Development Alternatives

### Free AI Options

Instead of OpenAI, you can use:

1. **Google Gemini** (free tier: 60 requests/min)
   ```bash

   GEMINI_API_KEY=your-key
   ```

2. **Ollama** (completely free, local)
   ```bash

   # Configure
   OLLAMA_MODEL=llama3.2
   ```

### Running Without Supabase

The app works offline-first:
- Data stored locally with AsyncStorage
- Syncs to Supabase when available
- All features functional without backend

## Troubleshooting

### Server won't start
- Check Node.js version: `node --version` (need >= 22.11.0)
- Check port isn't in use: `lsof -i :8787`
- Verify .env file exists

### AI plans not generating
- Verify OPENAI_API_KEY is set
- Check OpenAI account has credits
- Look at server logs for error messages
- Fallback mode should still work

### Database errors
- Verify SUPABASE_URL and SUPABASE_ANON_KEY
- Run schema.sql in Supabase SQL Editor
- Check RLS policies are enabled

## Security Notes

- Never commit `.env` file
- SUPABASE_ANON_KEY is safe for client-side use
- Keep SUPABASE_SERVICE_ROLE_KEY secret (not used by this app)
- RLS policies protect user data
